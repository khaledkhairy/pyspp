# Context Transfer Document: Spherical Parameterization Chain Vertex Issue

## Problem Statement

**Current Issue:** Patches 5 and 7 have discrepancies between border vertices and chain vertices:
- **Patch 5:** 79 border vertices, but only 63 in chains (79.7% coverage), missing 16 vertices
- **Patch 7:** 63 border vertices, but only 60 in chains, missing 3 vertices

**Requirement:** Zero discrepancy - ALL border vertices must be included in edge chains for correct parameterization.

## Root Cause Analysis

The refined edge chain generation in `patch_info_gen.py` (section [15]) extracts edge chains from `PM['OUT_chain'][pix]` (generated by `border2chain`) by finding segments between consecutive key vertices. The issue can occur due to:

1. **`border2chain` incompleteness:** The function may not return all border vertices if:
   - The border chain is disconnected
   - The walking algorithm gets stuck
   - Starting point selection is poor

2. **Key vertex gaps:** If border vertices appear in segments that don't have keys at both ends, they might be missed during extraction.

3. **Chain extraction logic:** The current logic walks from one key to the next, but if keys are missing or incorrectly identified, segments can be skipped.

## Data Structures

### `PM` (Patch Mesh Structure)
```python
PM = {
    'P': [list of patch data],  # PM['P'][pix][0] = patch mesh
    'OUT_chain': {pix: array},  # Ordered border chain from border2chain
    'patch': {  # Refined edge chains (MORE RELIABLE)
        pix: {
            'key_dat': array,  # Key vertex pairs [currkey, nextkey]
            'edge_dat': [array, ...]  # Edge chains between keys
        }
    },
    'edge_dat': [array, ...],  # Initial edge chains (less reliable)
    'keys': array,  # Key vertices [patch_idx, vertex_idx, face_idx]
    'sentinels': array,  # Sentinel vertices for each edge
    'Edges': array,  # Patch-level edges [pix1, pix2]
    ...
}
```

## Key Concepts

- **Border vertices:** Vertices on patch boundaries (marked by `border_vertex` flag)
- **Chain vertices:** Vertices in edge chains (used for boundary value interpolation)
- **Key vertices:** Vertices at patch junctions (where 3+ patches meet)
- **Sentinel vertices:** Start/end points of edge chains
- **Edge chains:** Ordered sequences of vertices along patch boundaries between sentinels
